# update-data — 把新 SRT 自動轉成市場重點簡報

## 觸發時機
使用者輸入下列關鍵字 (任一):
- 「更新資料」「更新」「處理新檔案」
- 「update data」「update」「process new files」

## 流程

### 1. 找出未處理的 SRT
```bash
ls "data/"      # 列出所有原始字幕
ls "report/"    # 列出已產出的報告
```
比對方式: 對 `data/` 中的每個 `.srt`, 推導出對應報告檔名 (見下方「命名規則」), 若 `report/<檔名>.pdf` 不存在則視為未處理。

### 2. 讀字幕並消化內容
對每個未處理的 SRT 用 Read 工具讀完整檔案。逐字稿通常包含:
- 節目名稱與日期
- 主持人開場 (節目時間、自我介紹)
- 主要議題討論 (1~6 個主題)
- 數據、專有名詞、人物觀點
- 結語

整理時抓:
- 美股/台股漲跌數據 (指數、點位、百分比)
- 個股財報亮點 (營收年增率、Capex、現金流)
- 央行/政策事件
- 主持人核心觀點與金句 (做成 quote 區塊)
- 該集 5~8 個 Takeaway

### 3. 產出 HTML 簡報

**樣式範本一律參考** `report/2026_4_30_早晨財經速解讀_市場重點簡報.html` (最新版本) 或 `report/EP657_市場重點簡報.html`。完整 CSS 直接複製過去 — **不要重新發明樣式**。

頁面結構 (12~16 頁為宜):
1. **Cover** — `<section class="slide cover active">` 節目名 + 副標 + 日期主持人
2. **目錄** — 條列本期重點議題
3. **市場數據頁** — 用 `<table class="index-table">` 呈現指數
4. **主題頁 × N** — 每個議題 1~2 頁, 用 `card` / `grid-2` / `grid-3` / `quote` 組合
5. **Takeaway** — 末頁總結, 用 `<span class="tag">` 分類

可用元件 (詳見既有 HTML):
- `.card` / `.card.warn` / `.card.good` / `.card.note` — 四色卡片
- `.grid-2` / `.grid-3` — 網格佈局
- `.quote` — 金句引用
- `.highlight` — 黃底螢光筆
- `.red` / `.green` — 漲跌顏色
- `.tag` / `.tag.warn` / `.tag.good` — 分類標籤
- `.index-table` — 指數表格

### 4. 命名規則
| 節目 | 檔名範例 |
|------|----------|
| 早晨財經速解讀 | `2026_4_30_早晨財經速解讀_市場重點簡報` |
| 股癌 | `EP657_市場重點簡報` |
| 台股達人秀 | `ep327_台股達人秀_市場重點簡報` |
| 其他 YouTube | `<日期或EP>_<節目名簡稱>_市場重點簡報` |

### 5. 轉 PDF + 輸出
```bash
bash "scripts/html_to_pdf.sh" \
  "report/<檔名>.html" \
  "report/<檔名>.pdf"
```

腳本會處理 OneDrive 權限問題 (先寫 Temp 再 mv)。

### 6. 股癌專屬：更新股票追蹤資料庫

**只有處理股癌（檔名 `EP<數字>` 開頭）時才執行這步。** 其他節目跳過。

從本集 SRT / 簡報內容抽取所有「具體被點名」的股票（個股 + ETF），略過：
- 純概念性的產業關鍵字（如「記憶體」「載板」「機器人」）
- 私人公司無法投資的（OpenAI、Anthropic 等）
- 廣告口播的贊助商（如修麗可、NordVPN）

對每檔股票整理：
- 代號（台股用數字代號 / 美股用 ticker / ETF 用代號）
- 中文 / 英文名
- 市場（台股 / 美股 / ETF / 美股 ETF）
- 本集觀點摘要（1～2 句，抓主持人對它的具體看法或事件）

執行（在專案根目錄）：
```bash
python scripts/build_gooaye_stocks.py --append "EP<集數>" "<日期 YYYY-MM-DD>" "<代號>|<名稱>|<市場>|<觀點>;<代號>|<名稱>|<市場>|<觀點>;..."
```

腳本會：
- 在 `report/股癌_股票追蹤.xlsx` 的「逐集記錄」分頁追加每筆
- 在「股票清單」分頁更新該股票的「最近提到 / 累計次數 / 觀點」（首次出現會新增列）

**範例**：
```bash
python scripts/build_gooaye_stocks.py --append "EP659" "2026-05-08" "2330|台積電|台股|本集觀點...;NVDA|NVIDIA|美股|本集觀點..."
```

### 7. 回報
告訴使用者:
- 處理了哪幾個檔案
- 每份報告的主要章節 (3~5 條)
- 檔案大小確認
- 若有股癌：本集新增 / 更新了多少檔股票到追蹤資料庫

## 注意事項
- **不要**改 `data/` 裡的原始 SRT
- **不要**把報告放到專案根目錄, 一律進 `report/`
- 既有的報告**不要重做**, 只處理新檔案
- 中文標題、引號使用全形, 數字百分比保持半形
- 一個節目對應一個 HTML + 一個 PDF, 兩個都要產
- 處理股癌時務必同步更新 `report/股癌_股票追蹤.xlsx`（見步驟 6）
